---
name: Expo Router typed routes regenerate on dev-server run
description: Why a freshly-added app/*.tsx route fails mobile typecheck until the Expo workflow restarts
---

Adding a new screen file under `artifacts/mobile/app/` (e.g. `app/paywall.tsx`) does NOT immediately make `router.push("/paywall")` typecheck. Expo Router's typed-routes union (`.expo/types/router.d.ts`) is generated by the Metro/dev server, not by `tsc`.

**Symptom:** `pnpm --filter @workspace/mobile run typecheck` fails with
`TS2345: Argument of type '"/paywall"' is not assignable to parameter of type 'RelativePathString | ... '`
for every `router.push`/`<Link href>` to the new route, even though the route file exists and is correct.

**Fix:** restart the `artifacts/mobile: expo` workflow (or otherwise let the Expo dev server run) so it regenerates the route types, then re-run typecheck. Prefer this over casting the href to `any`/`Href`.

**Why:** typed routes are derived from the filesystem at dev-server start; the type artifact is stale until then. Order matters when finishing mobile work: restart the workflow before the final typecheck.
